# Release Notes: v2.5.0

## 🚀 Megasheet Analytics Engine & API Hardening

**Summary:** 
This release introduces offline high-performance extraction capabilities for phase analytics, entirely bypassing browser memory limits. It additionally patches critical Auth restoration loopholes for FYERS accounts and vastly improves frontend gap-sync observability.

---

### ✨ What’s New (Features & Enhancements)

- **Background CLI Batch Engine (`batch_export.py`)**: 
  Designed a background loop to synchronously coordinate with the local Database and the Broker. Eliminates massive 5-year history browser timeouts for 200+ symbol watchlists.

- **Megasheet CSV Algorithm**: 
  Reworked frontend export utilities to transpose highly complex nested Regime profiles (Microstructure, Hit Probabilities, and Phase Extreemes) into a universally readable vertically stacked Megasheet CSV. 

- **Watchlist Screener Parity**: 
  Wired the Dashboard Screener Table directly to live backend microstructure datasets (rather than prototype endpoints). It accurately surfaces `Overall Efficiency`, `Max Spikes`, and `Primary Shapes`. 

- **DB Cache "Smart Sync" Introspection**: 
  Integrated gap-detection algorithms straight into the UI layer (`sync_details`). Visual indicators via terminal clearly differentiate between a zero-cost local `[CACHE HIT]` and an actual `[GAP SYNC]` payload.

---

### 🐛 Bug Fixes

- **Dashboard Aggregate Hard Crash**: 
  Rewrote structurally malformed React ternary JSX segments in `PhaseAnalyticsDashboard.jsx` allowing smooth un-crashing swaps between charts and the screener matrix.

- **Broker Authenticator Fast Paths**: 
  Hardened `auth.py`. Intercepted silent uninitialised `FYERS/ZERODHA` adapter exceptions inside `/profile` by aggressively checking local runtime memory state before attempting slower, deeper decryption routines.
